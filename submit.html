Submission Process

This is a community database of Bot automation codes.

Contributions to the file data/records.json, in which we list bot codes, are welcome!

This website as well as the list of bots displayed here are being version controlled using GitHub on the botcity-dev/bot-repository repository.


Prerequisites

In order to contribute you code you will need to have a GitHub Account.
If you don't have an account yet, click here and create one.


Keywords (Schema)

In order to parse the table efficiently, we apply a common schema.
Please use the following keywords for each entry in data/records.json.

Schema

Key Required Description
name Yes Name of the Bot
description_short Yes Short description (<= 300 characters)
description Yes Long description
tags Yes Keyword (tags): list of tags
programming_language Yes Programming language being used. E.g. python, java, c#
technology Yes Technologies used: list of strings. E.g. ["botcity", "robotframework"]
OS No List for: "linux", "macos", "windows"
thumbnail_url Yes An URL to the thumbail. Please use imgur.
youtube_video No An URL to a YouTube video
youtube_width No Width in pixels
youtube_height No Height in pixels
author_name No Name of the author or authors
author_site No An URL to the author profile
repository_name No Repository slug. E.g. "botcity-dev/repo"
repository_url Yes An URL to the code repository
company_name No Name of the company
company_site No An URL to the company website
license Yes The SPDX license identifier or "freeware", "proprietary", "unknown", as appropriate
upload_date Yes Date of publication in the dd/mm/yyyy format.

Contributing a Code

To contribute a code you will need to create a Pull Request with your changes to the data/records.json.
If you are already familiar with the pull request workflow just go ahead and submit a PR with your changes.
Otherwise please follow the instructions below.

1. Edit the data/records.json file:

  • Click on this link to edit the file.

  • Add your entry following the schema information above.

2. Proposing Changes

  • After making your changes, scroll down and fill the Propose changes box.

  • Click on the Propose changes button.

  • Review your changes and click on the Create pull request button.
    Doing so will create a Pull request with your code contribution. Learn about pull requests.

3. Review and Accepting Changes

Once your pull request is in, an automated process will validate the contributed code against the schema and our team will merge it into the codebase.

In case changes are required our team will reach out to you via the pull request using comments.
